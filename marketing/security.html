Security

Last Updated: January 1, 2026

OrenGen.io maintains enterprise-grade security measures to protect your data and ensure the integrity of our AI systems.

Security First: We implement military-grade encryption, SOC2 Type II compliance, ISO 27001 certification, and continuous security monitoring.

1. Security Overview

At OrenGen.io, security is not an afterthought—it's built into every layer of our infrastructure and development process. We employ:

  • Defense-in-depth architecture
  • Zero-trust security model
  • Continuous security monitoring
  • Regular penetration testing
  • Industry-leading compliance certifications

2. Data Encryption

2.1 Encryption in Transit

  • TLS 1.3: All data transmitted over networks uses TLS 1.3 encryption
  • Perfect Forward Secrecy: Session keys cannot be compromised even if long-term secrets are breached
  • Certificate Pinning: Additional protection against man-in-the-middle attacks
  • HSTS Enabled: HTTP Strict Transport Security forces HTTPS connections

2.2 Encryption at Rest

  • AES-256: Military-grade encryption for all stored data
  • Database Encryption: Full database encryption with rotating keys
  • File Storage: All files encrypted with unique per-file keys
  • Backup Encryption: Encrypted backups with separate key management

2.3 Key Management

  • AWS KMS: Hardware Security Modules (HSM) for key storage
  • Key Rotation: Automatic 90-day key rotation
  • Separation of Duties: No single person has access to all keys
  • Audit Logging: All key usage logged and monitored

3. Infrastructure Security

3.1 Cloud Infrastructure

  • Multi-Cloud Architecture: Primary: AWS, Backup: Google Cloud, Azure
  • Geographic Redundancy: Data replicated across multiple regions
  • DDoS Protection: Cloudflare Enterprise with 172 Tbps capacity
  • WAF (Web Application Firewall): OWASP Top 10 protection

3.2 Network Security

  • VPC Isolation: Private virtual networks for customer data
  • Network Segmentation: Separate networks for different security zones
  • Intrusion Detection: Real-time IDS/IPS monitoring
  • Zero Trust Architecture: No implicit trust within network perimeter

3.3 Server Security

  • Hardened Images: CIS Benchmark compliant server configurations
  • Automatic Patching: Security patches applied within 48 hours
  • Immutable Infrastructure: Servers replaced, not updated
  • Host-Based Firewalls: Restrictive firewall rules on all servers

4. Application Security

4.1 Secure Development Lifecycle

  • Security by Design: Security requirements defined at project start
  • Code Review: All code reviewed by security team before deployment
  • Static Analysis: Automated scanning for vulnerabilities (SAST)
  • Dynamic Analysis: Runtime security testing (DAST)

4.2 Authentication & Access Control

  • Multi-Factor Authentication (MFA): Available for all accounts, required for admins
  • OAuth 2.0: Industry-standard authentication protocol
  • SSO Support: SAML 2.0 single sign-on for enterprise customers
  • Role-Based Access Control (RBAC): Granular permission management
  • Session Management: Secure session tokens with automatic expiration

4.3 API Security

  • API Keys: Unique keys for each application and environment
  • Rate Limiting: Protection against abuse and DoS attacks
  • Input Validation: All inputs sanitized and validated
  • Output Encoding: Protection against injection attacks

5. Data Protection

5.1 Data Classification

  • Public: No restrictions (marketing materials)
  • Internal: OrenGen employees only
  • Confidential: Customer data, requires encryption
  • Restricted: PII, PHI, financial data - highest security level

5.2 Data Isolation

  • Multi-Tenancy: Logical isolation between customer accounts
  • Database Isolation: Separate schemas or databases for enterprise customers
  • No Cross-Contamination: Data from one customer never mixes with another

5.3 Data Retention & Deletion

  • Retention Policies: Data kept only as long as necessary
  • Secure Deletion: Cryptographic erasure and overwriting
  • Backup Purging: Data removed from all backups within 90 days
  • Right to Deletion: Customer data deleted within 30 days of request

6. Monitoring & Incident Response

6.1 24/7 Security Monitoring

  • SOC (Security Operations Center): 24/7 monitoring team
  • SIEM: Security Information and Event Management system
  • Log Aggregation: Centralized logging from all systems
  • Anomaly Detection: AI-powered threat detection

6.2 Incident Response

  • Incident Response Plan: Documented procedures for all scenarios
  • Response Time: Critical incidents acknowledged within 15 minutes
  • Escalation Procedures: Clear escalation paths for all severity levels
  • Post-Incident Review: Root cause analysis and remediation for all incidents

6.3 Breach Notification

  • 72-Hour Notification: Customers notified within 72 hours of confirmed breach
  • Transparency: Full disclosure of incident details
  • Regulatory Compliance: Reporting to authorities as required by law

7. Compliance & Auditing

7.1 Certifications

  • SOC 2 Type II: Annual audit by Deloitte & Touche
  • ISO 27001:2022: Information Security Management System
  • ISO 27018: Cloud Privacy Protection
  • PCI DSS: Payment Card Industry Data Security Standard (Level 1)

7.2 Compliance Standards

  • GDPR: General Data Protection Regulation (EU)
  • CCPA: California Consumer Privacy Act
  • HIPAA: Healthcare data protection (for healthcare customers)
  • FERPA: Educational data protection (for education customers)

7.3 Regular Auditing

  • Internal Audits: Quarterly security control reviews
  • External Audits: Annual third-party penetration testing
  • Compliance Audits: Annual SOC 2 and ISO 27001 audits
  • Vendor Audits: Annual security assessments of third-party vendors

8. Physical Security

8.1 Data Center Security

  • Tier III+ Facilities: Enterprise-grade data centers
  • 24/7 Security Guards: Physical security personnel
  • Biometric Access: Fingerprint and retina scanning
  • Video Surveillance: 24/7 recording with 90-day retention
  • Mantrap Entry: Two-factor authentication for physical access

8.2 Office Security

  • Badge Access: Electronic access control systems
  • Visitor Management: All visitors logged and escorted
  • Clean Desk Policy: No sensitive information left unattended
  • Secure Disposal: Shredding and degaussing of all media

9. Employee Security

9.1 Background Checks

  • Criminal background checks for all employees
  • Education and employment verification
  • Credit checks for financial roles
  • Ongoing monitoring for security-sensitive positions

9.2 Security Training

  • Onboarding Training: Security awareness for all new hires
  • Annual Training: Mandatory security refresher courses
  • Phishing Simulations: Quarterly phishing tests
  • Role-Specific Training: Advanced training for developers and admins

9.3 Access Management

  • Principle of Least Privilege: Minimum necessary access only
  • Regular Reviews: Quarterly access audits
  • Immediate Revocation: Access removed upon termination
  • Privileged Access Management: Separate accounts for admin tasks

10. Responsible Disclosure

10.1 Bug Bounty Program

  • Platform: HackerOne
  • Rewards: $100 - $10,000 per vulnerability
  • Scope: All OrenGen.io domains and APIs
  • Response Time: Initial response within 24 hours

10.2 Reporting Vulnerabilities

Found a Security Issue?

  1. Day 0: You report the vulnerability
  2. Day 1: We acknowledge receipt and begin investigation
  3. Day 7: We provide initial assessment and timeline

  5. Day 90: Public disclosure (coordinated)

11. Third-Party Security

11.1 Vendor Risk Management

  • Security assessments before vendor onboarding
  • Annual security questionnaires
  • SOC 2 reports required for critical vendors
  • Right-to-audit clauses in all contracts

11.2 Approved Vendors

Our critical vendors maintain equivalent or higher security standards:

  • Cloud Infrastructure: AWS (SOC 2, ISO 27001, FedRAMP)
  • CDN: Cloudflare (SOC 2, ISO 27001)
  • Payment Processing: Stripe (PCI DSS Level 1)
